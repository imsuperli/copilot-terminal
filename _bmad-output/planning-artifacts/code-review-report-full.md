## 1. 架构层面问题

### 🔴 1.1 主进程职责过重

**严重程度**: 高  
**文件**: src/main/index.ts (936 行)

**问题**: 主进程文件包含太多职责，违反单一职责原则

**建议**: 拆分为多个模块 (WindowManager, IPCRegistry, StateManagers)

### 🔴 1.2 重复的窗口/窗格管理逻辑

**严重程度**: 高  
**文件**: main/index.ts, CardGrid.tsx, TerminalView.tsx

**问题**: 窗口启动、暂停、关闭逻辑在多处重复实现

**建议**: 创建统一的 WindowPaneManager 服务

---

## 2. 代码重复问题

### 🟡 2.1 重复的状态更新逻辑

**文件**: windowStore.ts

**问题**: 每个 action 都重复调用 triggerAutoSave

**建议**: 使用 Zustand middleware 或 subscribe 机制

### 🟡 2.2 重复的错误处理模式

**文件**: 多个组件

**问题**: 错误处理逻辑分散，不统一

**建议**: 创建统一的 ErrorHandler 工具类

---

## 3. 类型安全问题

### 🟡 3.1 IPC 通信缺少类型约束

**文件**: preload/index.ts

**问题**: IPC 事件使用 unknown 类型，缺少类型检查

**建议**: 定义完整的 IPCEvents 接口

---

## 4. 逻辑复杂度问题

### 🟡 4.1 窗口关闭逻辑过于复杂

**文件**: main/index.ts (151-264 行)

**问题**: 包含太多嵌套和分支，难以理解和测试

**建议**: 拆分为 WindowCloseHandler 类

### 🟡 4.2 布局树操作逻辑复杂

**文件**: layoutHelpers.ts

**问题**: 递归逻辑缺少注释，难以理解

**建议**: 添加详细注释和示例

---

## 5. 状态管理问题

### 🟡 5.1 状态同步问题

**问题**: 主进程和渲染进程状态可能不同步

**建议**: 明确单一数据源，添加状态验证机制
