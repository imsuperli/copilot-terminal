# Product Brief: ausome-terminal

## Executive Summary

ausome-terminal 是一款面向 AI CLI 工具开发者的桌面窗口管理软件。随着 Claude Code、OpenCode 等 AI CLI 工具的爆发，开发者进入"一人多任务并行"模式，每天需同时管理 10+ 个 CLI 窗口。现有终端工具缺乏任务级状态管理和会话持久化能力，导致窗口混乱、状态不可见、上下文丢失、操作繁琐。ausome-terminal 不替代 Windows Terminal，而是增强它——在统一视图下管理所有 CLI 任务窗口，提供实时状态追踪、会话持久化和智能提醒，让开发者清晰掌控并行任务全局。

## Core Vision

### Problem Statement

随着 AI CLI 工具（Claude Code、OpenCode 等）的爆发式增长，开发者正在进入"一人多任务并行"的新工作模式。一个开发者每天需要同时管理 10+ 个 CLI 窗口，分布在不同项目、不同工作目录中。当前的核心痛点：

- **窗口混乱**：多标签/多窗口无法区分每个窗口在做什么任务，找不到需要操作的窗口
- **状态不可见**：需要人工轮询每个窗口，才能知道哪个在等待输入、哪个在运行、哪个已完成
- **上下文丢失**：每天重新打开 10+ 个窗口，重复配置工作目录和环境变量，无法延续昨天的工作状态
- **操作繁琐**：来回切换路径、重复动作，容易出错

### Problem Impact

- 开发者每天浪费大量时间在窗口切换和状态确认上
- 人工轮询窗口导致注意力碎片化，打断心流状态
- 重复配置工作环境降低工作效率，增加出错概率
- 无法有效利用 AI CLI 工具的并行能力，限制了生产力提升

### Why Existing Solutions Fall Short

现有工具（如 Windows Terminal）提供了基础的标签页和分屏功能，但存在根本性不足：

- 操作简陋，无法记住工程状态
- 标签页无法展示任务级别的信息（工作目录、模型、状态）
- 不支持会话持久化，每天需要重新打开和配置所有窗口
- 没有状态追踪和主动提醒机制，需要人工轮询
- 市场上尚无针对 AI CLI 工具多任务并行场景的专用解决方案

### Proposed Solution

ausome-terminal 是一款桌面窗口管理软件，核心能力：

1. **统一视图管理** — 所有窗口一目了然，每个窗口显示工作目录、使用模型、运行状态，告别盲目切换
2. **状态追踪与提醒** — 实时监控窗口状态（等待输入/运行中/已完成/出错），主动提醒用户介入，无需人工轮询
3. **会话保存与恢复** — 关闭软件后，下次打开恢复所有窗口状态，继续昨天的任务，零重复配置
4. **灵活的终端支持** — 不是 shell 实现，而是 shell 工具的增强包装层，支持用户自选终端工具组合（Windows Terminal + pwsh7 等）
5. **自定义配置** — 支持自定义环境变量、启动命令（如 claude --xxx），适应不同开发者的工作习惯

### Key Differentiators

- **定位差异**：不替代终端，而是增强终端，作为 shell 工具的包装层
- **时机优势**：AI CLI 工具爆发期，开发者从"单任务"进入"多任务并行"模式，市场空白
- **用户洞察**：来自重度用户的真实痛点，非假设性需求
- **可扩展性**：支持不同终端工具组合，不绑定特定终端

### Target Users

使用 AI CLI 工具（Claude Code、OpenCode 等）的开发者，日常需要同时管理多个并行任务窗口。

### Success Criteria

用户每天打开 ausome-terminal，看到昨天的所有工作窗口，快速识别未完成任务，点击进入继续工作，完成后标记为已完成（绿色），整个过程流畅、清晰、零重复操作。

## MVP Scope

### Core Features

**1. 统一视图管理**
- 在单一界面展示所有 CLI 任务窗口
- 每个窗口显示：工作目录、使用的模型、运行状态、窗口简称/别名
- 清晰的视觉区分（颜色编码）：运行中、等待输入、已完成、出错

**2. 窗口操作与管理**
- 点击窗口卡片进入对应的 CLI 窗口（保持 CLI 所有功能和操作不变）
- 新建任务窗口：配置工作目录、启动命令、环境变量
- 手动标记窗口状态（如标记为已完成/绿色）
- 关闭/删除窗口

**3. 会话持久化**
- 保存所有窗口配置（工作目录、环境变量、启动命令、终端工具选择）
- 软件关闭后，下次打开自动恢复所有窗口状态
- 零重复配置，即开即用

**4. 智能状态追踪**
- 自动检测窗口状态变化（通过窗口标题变化等技术手段）
- 实时显示哪些窗口正在运行、哪些已停止、哪些需要用户介入
- 主动提醒用户需要操作的窗口（视觉提示或通知）

**5. 灵活的终端支持**
- 支持用户自选终端工具组合（优先支持 Windows Terminal + pwsh7）
- 每个窗口可配置不同的终端工具
- 自定义环境变量和启动命令（如 `claude --model opus`）

**6. 平台支持**
- MVP 阶段专注 Windows 平台
- 架构设计考虑跨平台扩展性，为未来 macOS/Linux 支持预留空间

### Out of Scope for MVP

- **跨平台支持**：macOS 和 Linux 版本延后到后续版本
- **内置 AI 功能**：如自动总结任务进度、智能推荐等，MVP 不包含
- **多用户协作**：共享窗口状态、团队协作功能不在 MVP 范围
- **任务依赖管理**：窗口间的任务依赖关系和编排，留待未来版本
- **高级自动化**：自动需求分析、并行开发编排等智能功能，属于长期愿景

### MVP Success Criteria

- **自用验证通过**：日常使用中完全替代原有的多窗口管理方式，连续使用 30 天无回退
- **核心价值实现**：零重复配置、即时进入工作状态、不再漏掉等待输入的窗口
- **稳定性达标**：管理 10+ 窗口时运行稳定，无崩溃或数据丢失
- **性能可接受**：窗口切换响应迅速（< 500ms），状态更新实时（< 1s）
