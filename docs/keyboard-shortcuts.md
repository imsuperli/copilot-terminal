# 键盘快捷键清单

## 全局快捷键（非终端输入焦点）

这些快捷键只在焦点不在 xterm 输入框时由应用处理：

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+Tab` | 打开快速切换面板 | 模糊搜索窗口名称或路径，支持键盘导航 |
| `Escape` | 关闭打开的面板 | 关闭快速切换面板，如果没有面板打开则传递给终端 |

## 终端内快捷键

这些快捷键由宿主层接管，其他按键默认透传给终端程序：

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+C` | 复制选中文本 | 如果有选中文本则复制，否则发送中断信号 |
| `Ctrl+V` | 粘贴剪贴板内容 | 本地终端直接粘贴文本；SSH 终端中若剪贴板是图片，则优先上传图片并提示远端路径 |
| 右键粘贴 | 粘贴剪贴板内容 | 在捕获阶段接管，避免 xterm helper textarea 被原生 paste 污染 |
| SSH 图片粘贴快捷键 | 上传图片到远端 | 仅 SSH pane 生效，具体组合键由设置决定 |

## 终端输入焦点下已释放的快捷键

当焦点位于 xterm 输入框时，这些按键不再由 Synapse 改写或拦截，而是优先透传给 CLI：

| 快捷键 | 之前的行为 | 当前行为 |
|--------|------------|----------|
| `Ctrl+Enter` / `Cmd+Enter` | 应用强制注入 `\n` | 交给终端程序自行解释 |
| `Ctrl+J` / `Shift+Enter` | 依赖终端程序能力 | 继续透传，由 CLI 自行解释 |
| `Ctrl+Tab` | 打开快速切换面板 | 终端输入焦点下透传给 CLI |
| `Ctrl+B` | 切换侧边栏 | 终端输入焦点下透传给 CLI |
| `Ctrl+1~9` | 切换到第 N 个窗口 | 终端输入焦点下透传给 CLI |

## 已释放的快捷键

| 快捷键 | 原功能 | 释放原因 |
|--------|--------|----------|
| `Ctrl+P` | 打开快速切换面板 | 用户要求释放，改用 Ctrl+Tab |
| `Ctrl+Shift+Tab` | 向后切换窗口（MRU 顺序） | TabSwitcher 功能已移除 |

## 快捷键优先级

1. **宿主层只处理必要能力**：粘贴、右键粘贴、SSH 图片上传这类浏览器/宿主相关能力优先。
2. **终端输入焦点优先透传**：`Ctrl+Enter`、`Ctrl+J`、`Shift+Enter`、`Ctrl+Tab`、`Ctrl+B`、`Ctrl+1~9` 等按键不再由应用重写。
3. **UI 面板只在已打开时消费 `Escape`**：没有面板打开时，`Escape` 继续传给终端。

## 注意事项

- 快捷键只在对应窗口激活时生效（`enabled: isActive`）
- xterm 初始化时会开启增强键盘协商能力，让终端程序自行选择 kitty keyboard / Win32 input mode 等协议。
- macOS 不会启用 Windows 专属的 PTY 兼容提示；`Option` / `Command` 行为仍按平台原生规则交给终端和 CLI。
- `Escape` 键的行为是智能的：有面板打开时关闭面板，否则传递给终端（如 vim 的退出）
