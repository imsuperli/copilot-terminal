## 安装和启动

### 方式一：下载安装包（推荐）

1. 访问 [Releases 页面](https://github.com/your-repo/copilot-terminal/releases)
2. 下载适合你操作系统的安装包：
   - Windows: `.exe` 安装程序或 `.zip` 压缩包
   - macOS: `.dmg` 镜像文件
   - Linux: `.AppImage` 或 `.deb`/`.rpm` 包
3. 安装并启动应用

### 方式二：从源码运行（开发者）

```bash
# 1. 克隆仓库
git clone https://github.com/your-repo/copilot-terminal.git
cd copilot-terminal

# 2. 安装依赖（注意：需要先准备 xterm.js 自定义包）
npm install

# 3. 启动开发环境
npm run dev
```

**重要提示**：从源码运行前，请先阅读 [xterm.js 自定义依赖约束](xterm-custom-package-constraint.md)。

## 基本概念

Copilot-Terminal 采用三层嵌套结构来组织终端会话：

```
窗口组（WindowGroup）
  └─ 窗口（Window）
      └─ 窗格（Pane）
```

### 窗格（Pane）

**定义**：最小的终端单元，对应一个 PTY 进程。

**特性**：
- 每个窗格运行一个独立的 shell 进程
- 拥有自己的状态（Running、WaitingForInput、Paused、Exited）
- 可以独立启动、暂停、停止

### 窗口（Window）

**定义**：包含一个或多个窗格的容器，支持分屏布局。

**特性**：
- 可以包含多个窗格（通过分割创建）
- 窗格之间可以左右或上下分割
- 支持嵌套分割（递归的树形布局结构）
- 窗口的状态由其所有窗格的状态决定

**状态计算规则**：
```
窗口状态 = 该窗口所有窗格中优先级最高的窗格状态
优先级：Running > WaitingForInput > Paused > Exited
```

### 窗口组（WindowGroup）

**定义**：包含多个窗口的逻辑分组，用于组织相关的终端会话。

**特性**：
- 可以包含多个窗口
- 支持批量操作（启动所有、暂停所有、归档、删除）
- 可以在专用的 GroupView 中以分屏方式显示所有窗口
- 窗口组的状态由其所有窗口的状态决定

**状态计算规则**：
```
窗口组状态 = 该窗口组所有窗口中优先级最高的窗口状态
而每个窗口的状态 = 该窗口所有窗格中优先级最高的窗格状态
优先级：Running > WaitingForInput > Paused > Exited
```

## 界面布局

Copilot-Terminal 提供两种主要视图：

### 统一视图（Unified View）

- **主页卡片视图**：以卡片形式展示所有窗口和窗口组
- **窗口卡片**：显示单个窗口的信息（名称、路径、Git 分支、状态、项目链接）
- **窗口组卡片**：显示窗口组的信息（名称、窗口数量、各窗口状态）
- **操作按钮**：启动、暂停、归档、删除等

### 终端视图（Terminal View）

- **全屏终端界面**：沉浸式终端操作体验
- **侧边栏**：显示所有窗口列表，支持快速切换
- **快速切换器**：`Ctrl+Tab` 唤出，支持模糊搜索
- **状态栏**：显示当前窗口信息和 Claude StatusLine

## 创建和管理窗口

### 创建新窗口

1. 在统一视图中，点击"新建窗口"按钮
2. 输入窗口名称和工作目录
3. 点击"创建"

### 启动窗口

- 窗口创建后默认处于暂停状态
- 点击窗口卡片上的"启动"按钮来启动 PTY 进程
- 或在终端视图中点击顶部工具栏的"启动"按钮

### 暂停窗口

- 点击"暂停"按钮来暂停 PTY 进程
- 暂停后可以再次启动恢复

### 归档窗口

- 点击"归档"按钮将窗口移至归档区
- 归档的窗口不会显示在主列表中
- 可以在归档区查看和恢复归档的窗口

### 删除窗口

- 点击"删除"按钮永久删除窗口
- 删除操作不可恢复，请谨慎操作

### 分割窗格

在终端视图中：
1. 右键点击窗格区域
2. 选择"横向分割"或"纵向分割"
3. 新窗格将在当前窗格旁边创建

## 创建和管理窗口组

### 创建窗口组

1. 在统一视图中，选择多个窗口（按住 Ctrl 点击）
2. 点击"创建窗口组"按钮
3. 输入窗口组名称
4. 选择布局方式（横向或纵向）
5. 点击"创建"

### 打开窗口组

- 点击窗口组卡片进入 GroupView
- GroupView 以分屏方式显示组内所有窗口
- 可以在各窗口之间自由切换和操作

### 批量操作

窗口组支持以下批量操作：

- **启动所有**：启动组内所有窗口的 PTY 进程
- **暂停所有**：暂停组内所有窗口的 PTY 进程
- **归档**：将整个窗口组移至归档区
- **删除**：删除整个窗口组（窗口本身保留）

### 窗口组状态图标

#### 主页卡片中的状态图标

- 显示**多个状态图标**，每个图标代表组内的一个窗口
- 每个图标显示该窗口的状态
- 图标左上角的数字角标表示该窗口的窗格数量

**示例**：
```
窗口组包含：
- 窗口A：3个窗格（2个Running，1个WaitingForInput）
  → 窗口A状态 = Running
- 窗口B：2个窗格（1个WaitingForInput，1个Paused）
  → 窗口B状态 = WaitingForInput

显示结果：[❤️₃] [⌨️₂]
```

#### 侧边栏中的状态图标

- 显示**单个状态图标**
- 显示窗口组的整体状态（所有窗口状态中优先级最高的）
- 图标左上角的数字角标表示窗口组内的窗口数量

**示例**：
```
窗口组包含：
- 窗口A状态 = Running
- 窗口B状态 = WaitingForInput

窗口组状态 = Running（优先级最高）
窗口数量 = 2

显示结果：[❤️₂]
```

## SSH 图片粘贴

当当前窗格是 SSH 会话时，使用系统粘贴快捷键会先检查本地剪贴板是否为图片。

- 如果剪贴板是图片，应用会先把图片上传到远端目录，再在本地顶部显示上传成功提示。
- 默认会把远端图片路径复制回本地剪贴板，便于你再手动粘贴到 `codex`、`claude`、shell 命令或编辑器里。
- 如果剪贴板不是图片，或者为空，则不会触发上传逻辑，仍然按普通文本粘贴处理。

### 可配置项

在“设置 → 高级设置 → SSH 终端”中可以配置：

- 是否启用 SSH 图片粘贴上传
- 图片保存位置：当前位置、临时缓存目录、自定义目录
- 上传成功后是否自动复制远端路径
- 最大图片大小（MB）

### 上传位置说明

- 当前位置：优先使用当前 SSH pane 的工作目录；若不可用，会自动回退到 `~` 或 `/tmp`
- 临时缓存目录：优先使用 `~/.cache/copilot-terminal/images`，失败时回退到 `/tmp`
- 自定义目录：使用你指定的远端目录；若目录不存在，应用会尝试远端执行 `mkdir -p`

### 失败与限制

- 超过“最大图片大小”限制时，应用不会上传，并显示明确错误提示
- SSH 连接异常、远端目录无权限、磁盘空间不足等情况会导致上传失败
- 当前实现只处理图片，不处理文件、程序对象或其他非图片剪贴板内容

## 窗口状态和图标说明

### 状态类型

| 状态 | 图标 | 含义 | 说明 |
| --- | --- | --- | --- |
| Running | ❤️ 心电图 | 进程正在运行 | 有输出产生，进程活跃 |
| WaitingForInput | ⌨️ 键盘 | 等待用户输入 | 进程空闲，等待命令 |
| Paused | ⏸️ 暂停 | 进程已暂停 | PTY 进程未启动或已暂停 |
| Exited | （待定） | 进程已退出 | PTY 进程已终止 |

### 状态优先级

当窗口或窗口组包含多个窗格/窗口时，显示优先级最高的状态：

```
Running > WaitingForInput > Paused > Exited
```

### 数字角标

- **窗口卡片**：显示该窗口的窗格数量
- **窗口组卡片（主页）**：每个图标显示对应窗口的窗格数量
- **窗口组卡片（侧边栏）**：显示窗口组内的窗口数量

## 项目链接配置

### 什么是项目链接

项目链接允许你在窗口卡片上显示快捷按钮，用于快速访问相关资源（代码仓库、文档、监控面板等）。

### 配置方法

在项目根目录（即终端工作目录）创建 `copilot.json` 文件：

```json
{
  "version": "1.0",
  "links": [
    {
      "name": "GitHub Repo",
      "url": "https://github.com/your-org/your-repo"
    },
    {
      "name": "Documentation",
      "url": "https://docs.example.com"
    },
    {
      "name": "Monitoring Dashboard",
      "url": "https://grafana.example.com/dashboard"
    }
  ]
}
```

### 字段说明

- `version`：配置文件版本，当前为 `"1.0"`
- `links`：链接数组
  - `name`：链接名称（必须全局唯一，显示在按钮上）
  - `url`：链接地址（必须是有效的 URL）

### 使用方法

1. 创建 `copilot.json` 文件
2. 重启窗口或重新加载应用
3. 窗口卡片上会显示链接按钮（最多显示 6 个）
4. 点击按钮在默认浏览器中打开链接

## 快捷导航

### 什么是快捷导航

快捷导航是一个全局面板，用于快速访问常用的网站和本地文件夹。

### 打开快捷导航

- **方法一**：双击 `Shift` 键
- **方法二**：点击主侧边栏的指南针图标

### 添加导航项

1. 打开设置面板（`Ctrl+,`）
2. 切换到"快捷导航"标签
3. 点击"添加项目"
4. 输入名称和 URL/路径
5. 点击"保存"

### 导航项类型

- **URL**：以 `http://` 或 `https://` 开头，点击在浏览器中打开
- **文件夹路径**：本地文件夹路径，点击在文件管理器中打开

### 自动检测

- 系统会自动检测输入的类型（URL 或文件夹）
- 自动从 URL 域名或文件夹路径提取名称
- 可以手动编辑名称

## 工作区保存和恢复

### 自动保存

- Copilot-Terminal 会自动保存工作区状态
- 保存间隔：5 秒（可配置）
- 保存位置：`%APPDATA%/copilot-terminal/workspace.json`（Windows）

### 保存内容

工作区文件包含：
- 所有窗口的配置（名称、路径、状态、窗格布局）
- 所有窗口组的配置（名称、布局、窗口列表）
- 当前活动窗口/窗口组
- MRU 列表（最近使用的窗口和窗口组）
- 侧边栏状态（展开/折叠、宽度）

### 崩溃恢复

- 应用崩溃或异常退出后，下次启动会自动恢复工作区
- 所有窗口和窗口组会恢复到崩溃前的状态
- PTY 进程不会自动启动，需要手动启动

### 手动备份

你可以手动备份工作区文件：
1. 找到 `%APPDATA%/copilot-terminal/workspace.json`
2. 复制到安全位置
3. 需要恢复时，替换当前的 `workspace.json` 文件

## 常见问题

### Q: 窗口创建后为什么不能立即使用？

A: 窗口创建后默认处于暂停状态，需要点击"启动"按钮来启动 PTY 进程。这是为了防止启动时资源消耗过大。

### Q: 如何在终端中输入多行命令？

A: 使用 `Ctrl+Enter` 或 `Shift+Enter` 插入换行符，而不是执行命令。

### Q: 快速切换器搜索不到窗口？

A: 确保窗口名称或路径包含你输入的关键字。快速切换器支持模糊搜索。

### Q: 项目链接不显示？

A: 检查以下几点：
1. `copilot.json` 文件是否在项目根目录（终端工作目录）
2. JSON 格式是否正确
3. 是否重启了窗口或重新加载了应用

### Q: 窗口组删除后窗口会消失吗？

A: 不会。删除窗口组只会删除分组关系，窗口本身会保留在窗口列表中。

### Q: 如何恢复归档的窗口？

A: 在统一视图中，滚动到归档区，点击窗口卡片上的"恢复"按钮。

### Q: 状态图标不更新？

A: 状态检测基于 PTY 输出分析，更新间隔为 500ms。如果长时间不更新，尝试重启窗口。

### Q: 如何调整侧边栏宽度？

A: 在终端视图中，将鼠标悬停在侧边栏右边缘，出现调整光标后拖动即可。宽度范围：150-400px。

### Q: 支持哪些操作系统？

A:
- Windows 10/11
- macOS 10.15+
- Linux（主流发行版）

### Q: 如何更改默认 Shell？

A: 在设置面板中可以配置默认 Shell。Windows 优先使用 PowerShell 7+，其次是 PowerShell 5.1，最后是 cmd.exe。
