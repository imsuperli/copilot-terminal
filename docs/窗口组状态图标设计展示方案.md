# 窗口组状态图标设计展示方案

## 1. 概念层次

本应用采用三层嵌套结构来组织终端会话：

```
窗口组（WindowGroup）
  └─ 窗口（Window）
      └─ 窗格（Pane）
```

### 1.1 窗格（Pane）

**定义**：最小的终端单元，对应一个 PTY 进程。

**特性**：

- 每个窗格运行一个独立的 shell 进程
- 拥有自己的状态（Running、WaitingForInput、Paused、Exited）
- 可以独立启动、暂停、停止

**状态类型**：

- `Running`：进程正在运行，有输出产生
- `WaitingForInput`：进程等待用户输入（空闲状态）
- `Paused`：进程已暂停
- `Exited`：进程已退出

### 1.2 窗口（Window）

**定义**：包含一个或多个窗格的容器，支持分屏布局。

**特性**：

- 可以包含多个窗格（通过 tmux 兼容层的 split-window 命令创建）
- 窗格之间可以左右或上下分割
- 支持嵌套分割（递归的树形布局结构）
- 窗口的状态由其所有窗格的状态决定

**状态计算规则**：

```
窗口状态 = 该窗口所有窗格中优先级最高的窗格状态
优先级：Running > WaitingForInput > Paused > Exited
```

**示例**：

- 窗口A有3个窗格：2个Running，1个WaitingForInput
  - → 窗口A的状态 = Running（优先级最高）
- 窗口B有2个窗格：1个WaitingForInput，1个Paused
  - → 窗口B的状态 = WaitingForInput（优先级最高）

### 1.3 窗口组（WindowGroup）

**定义**：包含多个窗口的逻辑分组，用于组织相关的终端会话。

**特性**：

- 可以包含多个窗口
- 支持批量操作（启动所有、暂停所有、归档、删除）
- 可以在专用的 GroupView 中以分屏方式显示所有窗口
- 窗口组的状态由其所有窗口的状态决定

**状态计算规则**：

```
窗口组状态 = 该窗口组所有窗口中优先级最高的窗口状态
而每个窗口的状态 = 该窗口所有窗格中优先级最高的窗格状态
优先级：Running > WaitingForInput > Paused > Exited
```

## 2. 状态图标映射

每种状态对应一个视觉图标：

| 状态 | 图标 | 含义 |
| --- | --- | --- |
| Running | 心电图  | 进程正在运行 |
| WaitingForInput | 键盘 ⌨️ | 等待用户输入 |
| Paused | 暂停 ⏸️ | 进程已暂停 |
| Exited | （待定） | 进程已退出 |

注意：logo图标和当前单窗格状态logo一致

## 3. 窗口组状态图标展示方案

窗口组的状态图标在两个不同的场景下展示，展示逻辑有所不同：

### 3.1 场景一：主页卡片（UnifiedView 中的 GroupCard）

**展示位置**：窗口组卡片的左上角或标题区域

**展示内容**：

- **多个状态图标**，窗口组有几个窗口就显示几个图标
- 每个图标代表窗口组内的一个窗口
- 每个图标显示该窗口的状态
- 每个图标的左上角显示数字角标，表示该窗口的窗格数量

**计算逻辑**：

```typescript
for (每个窗口 in 窗口组) {
  窗口状态 = 该窗口所有窗格中优先级最高的状态
  窗格数量 = 该窗口的窗格总数
  显示: [状态图标 + 数字角标(窗格数量)]
}
```

**示例 1**：

```
窗口组包含：
- 窗口A：3个窗格（2个Running，1个WaitingForInput）
  → 窗口A状态 = Running
- 窗口B：2个窗格（1个WaitingForInput，1个Paused）
  → 窗口B状态 = WaitingForInput

显示结果：
[❤️₃] [⌨️₂]
```

**示例 2**：

```
窗口组包含：
- 窗口A：1个窗格（Running）
  → 窗口A状态 = Running
- 窗口B：4个窗格（3个Running，1个Paused）
  → 窗口B状态 = Running
- 窗口C：2个窗格（1个WaitingForInput，1个Exited）
  → 窗口C状态 = WaitingForInput

显示结果：
[❤️₁] [❤️₄] [⌨️₂]
```

### 3.2 场景二：终端界面侧边栏（GroupView 中的 Sidebar）

**展示位置**：GroupView 左侧边栏的窗口组标题区域

**展示内容**：

- **单个状态图标**
- 显示窗口组的整体状态（所有窗口状态中优先级最高的）
- 左上角显示数字角标，表示窗口组内的窗口数量

**计算逻辑**：

```typescript
窗口状态列表 = []
for (每个窗口 in 窗口组) {
  窗口状态 = 该窗口所有窗格中优先级最高的状态
  窗口状态列表.push(窗口状态)
}
窗口组状态 = 窗口状态列表中优先级最高的状态
窗口数量 = 窗口组内的窗口总数
显示: [状态图标 + 数字角标(窗口数量)]
```

**示例 1**（同上）：

```
窗口组包含：
- 窗口A状态 = Running
- 窗口B状态 = WaitingForInput

窗口组状态 = Running（优先级最高）
窗口数量 = 2

显示结果：
[❤️₂]
```

**示例 2**（同上）：

```
窗口组包含：
- 窗口A状态 = Running
- 窗口B状态 = Running
- 窗口C状态 = WaitingForInput

窗口组状态 = Running（优先级最高）
窗口数量 = 3

显示结果：
[❤️₃]
```

## 4. 两种场景的对比

| 维度 | 主页卡片（GroupCard） | 侧边栏（GroupView Sidebar） |
| --- | --- | --- |
| 图标数量 | 多个（= 窗口数量） | 单个 |
| 图标含义 | 每个窗口的状态 | 窗口组的整体状态 |
| 数字角标 | 该窗口的窗格数量 | 窗口组的窗口数量 |
| 信息粒度 | 细粒度（窗口级别） | 粗粒度（窗口组级别） |
| 使用场景 | 用户浏览所有窗口组，需要快速了解每个窗口组内各窗口的状态 | 用户已进入窗口组，需要了解整体状态和窗口数量 |

## 5. 实现要点

### 5.1 状态计算函数

需要实现以下工具函数：

```typescript
// 计算窗格的优先级
function getStatusPriority(status: WindowStatus): number {
  const priorities = {
    Running: 4,
    WaitingForInput: 3,
    Paused: 2,
    Exited: 1
  }
  return priorities[status] || 0
}

// 计算窗口的状态（基于窗格优先级）
function getWindowStatus(window: Window): WindowStatus {
  if (!window.panes || window.panes.length === 0) {
    return 'Exited'
  }

  let highestStatus = window.panes[0].status
  let highestPriority = getStatusPriority(highestStatus)

  for (const pane of window.panes) {
    const priority = getStatusPriority(pane.status)
    if (priority > highestPriority) {
      highestStatus = pane.status
      highestPriority = priority
    }
  }

  return highestStatus
}

// 计算窗口组的状态（基于窗口优先级）
function getGroupStatus(group: WindowGroup, windows: Window[]): WindowStatus {
  const groupWindows = windows.filter(w =>
    isWindowInGroup(group.layout, w.id)
  )

  if (groupWindows.length === 0) {
    return 'Exited'
  }

  let highestStatus = getWindowStatus(groupWindows[0])
  let highestPriority = getStatusPriority(highestStatus)

  for (const window of groupWindows) {
    const windowStatus = getWindowStatus(window)
    const priority = getStatusPriority(windowStatus)
    if (priority > highestPriority) {
      highestStatus = windowStatus
      highestPriority = priority
    }
  }

  return highestStatus
}
```

### 5.2 UI 组件

需要创建以下组件：

**StatusIconWithBadge**：

- 接收 `status: WindowStatus` 和 `count: number` 参数
- 渲染状态图标 + 左上角数字角标
- 可复用于两种场景

**GroupStatusIcons**（用于 GroupCard）：

- 接收 `group: WindowGroup` 和 `windows: Window[]` 参数
- 遍历窗口组内的所有窗口
- 为每个窗口渲染一个 `StatusIconWithBadge`
- 水平排列多个图标

**GroupStatusIcon**（用于 GroupView Sidebar）：

- 接收 `group: WindowGroup` 和 `windows: Window[]` 参数
- 计算窗口组的整体状态
- 渲染单个 `StatusIconWithBadge`

### 5.3 数据流

```
1. windowStore 维护 windows 和 groups 数组
2. GroupCard 组件接收 group 对象
3. GroupCard 从 windowStore 获取所有 windows
4. GroupCard 调用 GroupStatusIcons 组件
5. GroupStatusIcons 遍历 group.layout，找到所有窗口 ID
6. 对每个窗口，计算状态和窗格数，渲染图标
```

## 6. 边界情况处理

### 6.1 空窗口组

- 窗口组不包含任何窗口
- 主页卡片：不显示任何状态图标
- 侧边栏：显示 Exited 状态 + 数字 0

### 6.2 窗口无窗格

- 窗口不包含任何窗格（理论上不应该发生）
- 该窗口的状态视为 Exited

### 6.3 已归档的窗口组

- 窗口组已归档，所有窗口也已归档
- 状态图标正常显示（基于归档前的状态）
- 或者统一显示为 Paused 状态

### 6.4 窗口被删除

- 窗口组引用的窗口已被删除
- 自动从窗口组布局中移除该窗口
- 重新计算状态图标

## 7. 视觉设计建议

### 7.1 图标尺寸

- 主页卡片：每个图标 24x24px，间距 4px
- 侧边栏：单个图标 20x20px

### 7.2 数字角标样式

- 位置：图标右上角
- 背景：半透明圆形背景
- 字体：10px，加粗
- 颜色：白色文字 + 深色背景

### 7.3 图标排列

- 主页卡片：水平排列，最多显示 5 个，超出显示 "+N"
- 侧边栏：单个图标，居中显示

## 8. 未来扩展

### 8.1 悬停提示

- 鼠标悬停在图标上时，显示详细信息
- 主页卡片：显示该窗口的名称、路径、窗格列表
- 侧边栏：显示窗口组内所有窗口的状态分布

### 8.2 点击交互

- 主页卡片：点击某个图标，直接打开该窗口的终端视图
- 侧边栏：点击图标，展开窗口列表

### 8.3 动画效果

- 状态变化时，图标平滑过渡
- 新增窗口时，图标淡入动画
