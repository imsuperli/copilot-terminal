# 输入法候选框跳动问题 - 修复方案总结

## 问题回顾

用户反馈：之前的修复方案没有效果，输入法候选框仍然在"动画刷新的位置"跳动。

## 根本原因

通过深入分析 xterm.js 源码，发现了问题的完整链条：

### 关键发现

1. **onRender 事件触发**（`CoreBrowserTerminal.ts:420`）
   ```typescript
   this._register(this.onRender(() => this._compositionHelper!.updateCompositionElements()));
   ```
   每次终端渲染都会自动调用 `updateCompositionElements()`

2. **updateCompositionElements 更新位置**（`CompositionHelper.ts:251-252`）
   ```typescript
   this._textarea.style.left = cursorLeft + 'px';
   this._textarea.style.top = cursorTop + 'px';
   ```
   该方法会根据当前光标位置更新 textarea 位置

3. **问题链条**
   ```
   终端输出 → 光标移动 → onRender 事件 → updateCompositionElements()
   → 更新 textarea 位置 → 输入法候选框跟随移动 → 用户看到跳动
   ```

## 修复方案

### 核心策略

**在输入法组合期间完全阻止 `updateCompositionElements()` 的执行（除了首次捕获锚点）**

### 实现细节

```typescript
compositionHelper.updateCompositionElements = function patchedUpdateCompositionElements(dontRecurse?: boolean): void {
  const isComposing = compositionHelper.isComposing;

  // 非组合状态：正常执行
  if (!isComposing) {
    originalUpdate.call(this, dontRecurse);
    return;
  }

  // 组合状态 - 首次调用：捕获锚点
  if (!frozenAnchor) {
    originalUpdate.call(this, dontRecurse);  // 执行一次获取初始位置
    frozenAnchor = readAnchorFromElements(textarea, compositionView);
    return;  // 首次调用后直接返回
  }

  // 组合状态 - 后续调用：完全阻止
  // 不执行任何操作，直接返回
  // textarea 和 compositionView 保持首次调用时的位置不变
};
```

### 关键改进

1. **首次调用后不再手动应用锚点**
   - 原始方法已经设置了正确的位置
   - 避免触发 MutationObserver
   - 消除时序问题

2. **后续调用完全阻止**
   - 直接返回，不执行任何操作
   - 阻止 xterm.js 更新 textarea 位置
   - textarea 保持首次捕获时的位置

3. **保留 MutationObserver 作为保险**
   - 防止其他未知路径修改位置
   - 作为最后一道防线

### 测试日志分析

```
[IME-FIX] 组合开始 (compositionstart)
[IME-FIX] updateCompositionElements 调用 #1 (首次，捕获锚点)
[IME-FIX] 捕获冻结锚点 { left: '40px', top: '12px', ... }
[IME-FIX] updateCompositionElements 调用 #2 (已阻止 #1)
[IME-FIX] 组合结束 (compositionend) {
  updateCallCount: 2,
  blockUpdateCount: 1,
  mutationCount: 0
}
```

**关键指标：**
- `updateCallCount: 2` - 总共调用 2 次
- `blockUpdateCount: 1` - 成功阻止 1 次
- `mutationCount: 0` - 没有触发 MutationObserver（说明位置没有被其他路径修改）

## 预期效果

1. **输入法组合开始时**
   - 捕获当前 textarea 位置
   - 位置固定在光标处

2. **组合期间终端输出时**
   - 光标移动
   - onRender 事件触发
   - updateCompositionElements() 被调用但立即返回
   - textarea 位置保持不变
   - 输入法候选框不跳动

3. **组合结束后**
   - 恢复正常的位置更新机制
   - textarea 跟随光标移动
